# unik_netmonitor

## 📋 Описание проекта

**Монитор школьной сети** — это легковесное, автономное приложение для мониторинга локальной сети, разработанное в рамках проекта по направлению **Blue Team**. Программа позволяет оперативно выявлять неизвестные устройства в сети и уведомлять администратора через Telegram.

### 🎯 Цель проекта

Разработка легковесной, автономной системы мониторинга локальной сети, позволяющей оперативно выявлять неизвестные устройства и уведомлять администратора.

## 🚀 Основные возможности

- 🔍 **Гибридное сканирование сети** — комбинация активного пробуждения устройств (ping sweep) и пассивного анализа ARP-таблицы
- 📱 **Мгновенные уведомления** — отправка оповещений в Telegram при обнаружении новых устройств
- 📊 **Графический интерфейс** — интуитивно понятный интерфейс на базе Tkinter
- ✅ **Белый список** — возможность добавления доверенных MAC-адресов
- 📁 **История подключений** — сохранение данных о всех обнаруженных устройствах
- ⏱️ **Фоновый мониторинг** — автоматическое сканирование сети каждые 60 секунд
- 📝 **Структурированное логирование** — запись событий в формате JSON Lines для последующего анализа

## 📦 Установка и запуск

### Требования

- **Операционная система**: Windows 7/8/10/11, Linux, macOS
- **Python**: 3.7 или выше
- **Библиотеки**: `requests`

### Установка зависимостей

```bash
pip install requests
```

### Настройка конфигурации

Создайте файл `config.ini` в той же директории, где находится программа:

```ini
[telegram]
bot_token = ваш_токен_бота
chat_id = ваш_chat_id

[network]
subnet_mask = /24
```

### Запуск программы

```bash
python school_network_monitor.py
```

### Сборка в исполняемый файл

```bash
py -m PyInstaller --onefile --windowed school_network_monitor.py
```

## 📖 Руководство пользователя

### 1. Начальная настройка

1. Запустите программу
2. Введите токен бота и chat_id (если используете Telegram)
3. Настройте параметры сети (при необходимости)

### 2. Сканирование сети

- Нажмите кнопку **«Сканировать сейчас»** для однократного сканирования
- Или **«Запустить фон»** для автоматического мониторинга каждые 60 секунд

### 3. Управление белым списком

- Выберите устройство в таблице
- Нажмите **«Добавить в белый список»**, чтобы добавить его в доверенные
- Или **«Удалить из белого списка»**, чтобы исключить

## 📊 Формат логирования

Программа использует формат **JSON Lines** (`.jsonl`) для логирования событий. Каждая строка файла — это валидный JSON-объект со следующими полями:

| Поле | Описание | Пример |
|------|----------|--------|
| `timestamp` | Время события в формате ISO 8601 | `"2026-03-14T15:30:45.123456+00:00"` |
| `level` | Уровень важности | `"INFO"`, `"WARNING"`, `"ERROR"` |
| `event_type` | Тип события | `"device_detected"`, `"scan_completed"` |
| `message` | Человекочитаемое сообщение | `"Обнаружено новое устройство"` |
| `mac` | MAC-адрес устройства | `"a0:b1:c2:d3:e4:f5"` |
| `ip` | IP-адрес устройства | `"192.168.1.42"` |

**Пример записи:**

```json
{"timestamp": "2026-03-14T15:30:45.123456+00:00", "level": "INFO", "event_type": "device_detected", "message": "Обнаружено новое устройство: a0:b1:c2:d3:e4:f5", "mac": "a0:b1:c2:d3:e4:f5", "ip": "192.168.1.42"}
```

## 🔒 Безопасность

- Программа **не требует прав администратора** для запуска
- Все данные хранятся **локально**, без передачи в облачные сервисы
- Токены и пароли **не отображаются** в интерфейсе в открытом виде
- Логирование ведётся в **отдельный файл**, не влияя на производительность
